# ROADMAP — PAM (стабильный план фаз)

> Источник визуального плана — `../implementation-plan.html` (открывать в браузере).
> Здесь — рабочая разбивка на шаги для GSD-процесса. Текущая позиция — в `STATE.md`.

## Продуктовое видение (уточнено 2026-06-02)

PAM = личный AI с **долгой памятью о тебе**. Два основных столпа + вспомогательный раздел.

**ОСНОВНОЕ (main):**
1. **Чат с долгой памятью** — главная страница, UI как у claude/chatgpt (сайдбар чатов + поле ввода).
   Пишешь прямо в PAM; AI отвечает (Groq→Claude), достаёт релевантный контекст из накопленной
   памяти (прошлые чаты + факты о тебе) через RAG → «помнит всё» и со временем строит модель тебя.
   → это **Phase 3** (память/факты) + **Phase 4** (чат), но чат становится **ГЛАВНЫМ** экраном.
2. **Личный лектор** — раздел: кидаешь PDF / YouTube-ссылку / статью → персональный мини-курс
   с тестами под твой уровень (не пересказ, а обучение; уровень берётся из памяти о тебе).
   → новый столп, **Phase 5**.

**ДОПОЛНИТЕЛЬНОЕ (раздел, бывший Phase 1):**
- **История / Импорт** — браузерное расширение тянет разговоры из ChatGPT/Claude/Gemini, чтобы
  засеять память прошлым (иначе ассистент стартует с нуля). Просмотр + полнотекстовый поиск.

**Общая инфраструктура (нужна всем трём):** Postgres+pgvector, чанкование+эмбеддинги (Phase 2),
слой памяти/фактов (Phase 3).

**Про парсер:** главному чату парсер НЕ нужен — разговоры рождаются внутри PAM и пишутся в БД
напрямую. Парсер разговоров (extension) остаётся только для вспомогательного «Импорта истории» —
по желанию. Лектору нужны ДРУГИЕ парсеры контента: PDF-текст, YouTube-транскрипт, статья→текст.

## Принципы
- Каждая фаза шиппится самостоятельно. Не тянуть Phase N+1 в Phase N.
- Один branch на фазу: `phase-2-rag`, `phase-3-memory`, `phase-4-chat`, `phase-5-lecturer`.
- Мелкие шаги → atomic commits → обновление `handoff.md`.
- **Значимые фичи → отдельная ветка с понятным именем + понятные коммиты** (предпочтение пользователя).

### Импорт истории — доводка раздела (бывший Phase 1)
- [x] Вынести список+поиск в раздел `/history`, главная → лендинг с разделами, нормальный nav-бар.
- [x] Диагностика полноты перехвата: лог `extracted N of M` + разбивка по ролям (chatgpt).
- [ ] **Полнота перехвата:** ChatGPT отдаёт дерево `mapping`; сейчас BFS по всем веткам. Решить: путь по `current_node` (точная видимая нить, верный порядок) vs BFS-надмножество (вся история, включая правки). Проверить ленивую подгрузку очень длинных чатов.
- [ ] **Избранное (manual)**: кнопка ★ на сообщении → снимок в отдельную таблицу `saved_messages`. ВАЖНО: нельзя хранить флаг на `messages` — они вайпятся при повторном захвате (UPSERT wipe-reinsert). Снимок переживает ре-ингест. Делается рано, без эмбеддингов.
- [ ] **Авто-«популярное» (auto)**: нейронка сама выделяет часто повторяющиеся/похожие ответы. Требует эмбеддинги из Phase 2 (по ним считается «похожесть» и частота тем). Делается вместе с/после RAG.
- [ ] **Mindmap тем** «что чаще всего спрашиваешь»: v1 — частотность ключевых слов; v2 — кластеризация вопросов по эмбеддингам (зависит от Phase 2).
- [ ] **Аналитика**: частота тем/вопросов во времени.
- Заметка: перехват идёт по JSON из `fetch`, НЕ по DOM/скроллу — порядок старых тем «вверху чата» в ChatGPT на полноту захвата не влияет (берём весь ответ сервера).
